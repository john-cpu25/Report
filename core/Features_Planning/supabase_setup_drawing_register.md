# Cấu hình Database Supabase cho Drawing Register & Issues

## Tình trạng: ✅ ĐÃ HOÀN TẤT

Bảng `NMK_DrawingRegister` và `NMK_Issue` đã được tạo trực tiếp trên Supabase.

---

## Bảng 1: `NMK_DrawingRegister`
Lưu toàn bộ dữ liệu Drawing Register dưới dạng JSON trong cột `attachments`.

```sql
create table public."NMK_DrawingRegister" (
  id uuid not null,
  project_id uuid null,
  attachments text null,
  created_at timestamp with time zone null,
  updated_at timestamp with time zone null,
  constraint issues_pkey primary key (id)
) TABLESPACE pg_default;
```

**Cấu trúc JSON trong `attachments`:**
```json
{
  "projectNo": "11092",
  "projectName": "FITZROY GASWORKS - PARCEL B",
  "projectAddress": "433 SMITH STREET FITZROY NORTH VIC 3068",
  "client": "ASSEMBLE",
  "dateColumns": ["23/05/25", "23/06/25", "01/08/25", ...],
  "drawings": [
    {
      "sheetNo": "S0000",
      "sheetName": "DRAWING INDEX",
      "currRev": "B",
      "revisions": { "23/06/25": "A", "12/09/25": "B" },
      "pdfFile": null,
      "cadFile": null
    }
  ]
}
```

## Bảng 2: `NMK_Issue`
Liên kết file PDF/CAD từ Google Drive với mỗi bản vẽ cụ thể (sheet + revision).

```sql
create table public."NMK_Issue" (
  id bigint generated by default as identity not null,
  created_at timestamp with time zone not null,
  updated_at timestamp with time zone null,
  project_id bigint null,
  sheet_number text null,
  sheet_name text null,
  rev_date text null,
  rev_value text null,
  url text null,
  constraint nmk_issue_pkey primary key (id)
) TABLESPACE pg_default;
```

**Mapping Logic:**
Khi hiển thị bảng Drawing Register trên UI, mỗi ô revision sẽ tìm kiếm record trong `NMK_Issue` khớp **4 khóa**:
1. `project_id` - Mã dự án
2. `sheet_number` - Số bản vẽ (VD: "S0000")
3. `rev_date` - Ngày phát hành (VD: "23/06/25")
4. `rev_value` - Phiên bản (VD: "A")

Nếu tìm thấy → hiển thị link **Direct Download** từ Google Drive.
Nếu không → hiển thị text revision bình thường.
